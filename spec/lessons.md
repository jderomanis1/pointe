# Pointe — Lessons

Durable, dated lessons from building and operating Pointe. Append-only;
newest entries at the top.

2026-06-06 — Platform-direction decisions need current verification, not training-data priors. OQ-006 (static hosting) is the second time a Sessions 1–3 deviation from the spec aged into being correct because the Cloudflare platform shifted under the spec — the first was OQ-005 (words-slug). In both cases the spec's premise went stale: here, "use Pages" predated Pages being converged into Workers (the migration guide flipped to Pages→Workers, greenfield scaffolding moved to Workers-with-assets, and per-branch preview deploys shipped on Workers). A recon-first check of the *current* platform state — not a recall from priors — is what kept us from spending launch-eve days migrating onto a de-emphasized path. Same discipline that caught the KV-cache miscall, the WS-handshake NAT-rotation behaviour, and the workerd time-manipulation pitfall. Rule: any platform-capability or platform-direction claim gets verified against current docs before it drives a decision. The lesson is NOT "trust the build over the spec" — it's "verify platform claims now, don't trust stale priors (the spec's or the model's)."

2026-06-06 — AI dependency outage; graceful-failure verified under real conditions. During the S10 a11y pass the Anthropic API returned 400 (org out of API credits). The graceful-failure path (Fix 06) held in the wild: the host saw an "AI unavailable" banner, voting continued uninterrupted, AA-1 held (voters saw nothing), no broken state persisted, and the feature restored cleanly the moment credits were refilled. Most systems claim graceful dependency-failure handling; this one was tested by an actual outage and held. Datapoint, not hypothetical.
